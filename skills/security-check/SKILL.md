# Security Check Skill

セキュリティ脆弱性を検出・防止するための強制チェックスキルです。
OWASP Top 10や業界標準のセキュリティベストプラクティスに基づいて検証します。

## 使用タイミング

- コード実装時（自動チェック）
- コードレビュー時（必須チェック）
- 依存パッケージ追加時
- 認証・認可機能の実装時

## チェック項目

### 1. 認証・認可

- [ ] 認証トークンの有効期限が適切に設定されているか
- [ ] パスワードが適切にハッシュ化されているか（bcrypt, argon2等）
- [ ] セッション管理が適切に実装されているか
- [ ] 権限チェックがすべての保護されたエンドポイントで実装されているか
- [ ] 認証情報がログに出力されていないか
- [ ] 多要素認証（MFA）が実装されているか（要件に応じて）

### 2. 入力検証・サニタイゼーション

- [ ] すべてのユーザー入力が検証されているか
- [ ] SQLインジェクション対策が実装されているか（プリペアドステートメント使用）
- [ ] XSS対策が実装されているか（エスケープ処理、CSP）
- [ ] コマンドインジェクション対策が実装されているか
- [ ] パストラバーサル対策が実装されているか
- [ ] ファイルアップロードの検証が実装されているか（拡張子、サイズ、MIMEタイプ）

### 3. 機密情報の取り扱い

- [ ] APIキー、パスワード、トークンがハードコードされていないか
- [ ] 環境変数またはシークレット管理サービスを使用しているか
- [ ] 機密情報がログに出力されていないか
- [ ] 機密情報がバージョン管理システムにコミットされていないか
- [ ] 暗号化が必要なデータが適切に暗号化されているか

### 4. 通信セキュリティ

- [ ] HTTPSが強制されているか（本番環境）
- [ ] TLS/SSL証明書の検証が適切に行われているか
- [ ] CORS設定が適切に実装されているか
- [ ] セキュアなヘッダーが設定されているか（HSTS, CSP, X-Frame-Options等）

### 5. 依存パッケージ

- [ ] 依存パッケージに既知の脆弱性がないか（npm audit, go mod, 等）
- [ ] 依存パッケージが最新のセキュアなバージョンか
- [ ] 不要な依存パッケージが含まれていないか

### 6. エラーハンドリング

- [ ] エラーメッセージに機密情報が含まれていないか
- [ ] スタックトレースが本番環境で公開されていないか
- [ ] 適切なエラーログが記録されているか

### 7. レート制限・DoS対策

- [ ] レート制限が実装されているか（APIエンドポイント）
- [ ] リクエストサイズの制限が設定されているか
- [ ] タイムアウトが適切に設定されているか

## 検出された脆弱性の対応

### 重大度レベル

1. **Critical（重大）**: 即座に修正が必要。マージ不可
2. **High（高）**: 優先的に修正が必要。マージ前に修正推奨
3. **Medium（中）**: 可能な限り早期に修正
4. **Low（低）**: 改善提案として記録

### 出力形式

```
## セキュリティチェック結果

### 🔴 Critical
- [脆弱性名]: [詳細と修正方法]

### 🟠 High
- [脆弱性名]: [詳細と修正方法]

### 🟡 Medium
- [脆弱性名]: [詳細と修正方法]

### 🔵 Low
- [脆弱性名]: [改善提案]
```

# Skills

このディレクトリには、Claude Codeで使用するスキルが含まれています。
スキルは「守らせたいこと」を強制するために使用され、コード品質、セキュリティ、ドキュメント同期などのチェックを自動化します。

## スキル一覧

### code-review
コードレビュー時に必須のチェック項目を強制実行するスキル。
実装品質、セキュリティ、パフォーマンス、保守性を包括的に検証します。

**使用タイミング**: `/review` コマンド実行時、プルリクエスト作成前

### security-check
セキュリティ脆弱性を検出・防止するための強制チェックスキル。
OWASP Top 10や業界標準のセキュリティベストプラクティスに基づいて検証します。

**使用タイミング**: コード実装時、コードレビュー時、依存パッケージ追加時

### documentation-sync
コード変更に伴うドキュメントの同期を強制するスキル。
API変更、新機能追加、設定変更時にドキュメントが適切に更新されているかを確認します。

**使用タイミング**: コード実装時、コードレビュー時、API変更時

### quality-gate
コード品質の最低基準を保証するための品質ゲートスキル。
マージ前に必須の品質チェックを実行し、基準を満たさないコードのマージを防止します。

**使用タイミング**: コードレビュー時、マージ前、CI/CDパイプライン

### git-workflow
Git操作の標準化を強制するスキル。
コミットメッセージ、ブランチ命名規則、PRの説明、マージ戦略などを統一し、チーム全体で一貫したGitワークフローを維持します。

**使用タイミング**: コミット作成時、ブランチ作成時、プルリクエスト作成時

### changelog-generator
CHANGELOGの自動生成・更新を支援するスキル。
コミット履歴やPR情報からCHANGELOGエントリを生成し、バージョン管理とリリースノート作成を自動化します。

**使用タイミング**: リリース前、バージョン更新時、マージ後

### dependency-audit
依存関係の包括的な監査を実行するスキル。
セキュリティ脆弱性、ライセンス、バージョン管理、不要な依存関係を検証し、プロジェクトの依存関係を健全に保ちます。

**使用タイミング**: 新規依存パッケージ追加時、定期的な監査時、リリース前

### api-compatibility
API互換性を検証するスキル。
破壊的変更の検出、バージョン管理、後方互換性の確認を行い、APIの安定性と互換性を保証します。

**使用タイミング**: API変更時、リリース前、バージョンアップ時

## スキルの構造

各スキルは以下の構造を持ちます：

```
skill-name/
└── SKILL.md    # スキルの定義とチェック項目
```

## スキルの使用方法

スキルは `.claude/common/skills/` に配置され、Claude Codeが自動的に読み込みます。
特定のスキルを明示的に使用する場合は、コマンド実行時にスキル名を指定できます。

### 例

```bash
# コードレビュー時に自動的にcode-reviewスキルが適用される
/review .claude/project/01-tasks/001-feature.md

# セキュリティチェックを明示的に実行
security-checkを適用してこのコードをチェックして
```

## スキルの追加

新しいスキルを追加する場合は、以下の手順に従ってください：

1. `skills/` ディレクトリに新しいディレクトリを作成
2. `SKILL.md` ファイルを作成し、YAML frontmatterとスキル定義を記述
3. README.md（このファイル）にスキルの説明を追加
4. 必要に応じてCHANGELOG.mdを更新

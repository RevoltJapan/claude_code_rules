# Code Review Skill

コードレビュー時に必須のチェック項目を強制実行するスキルです。
実装品質、セキュリティ、パフォーマンス、保守性を包括的に検証します。

## 使用タイミング

- `/review` コマンド実行時
- プルリクエスト作成前の自己レビュー
- コード品質の最終確認時

## 必須チェック項目

### 1. コード品質

- [ ] 命名規則が一貫しているか（プロジェクトの規約に準拠）
- [ ] 関数/メソッドが単一責任の原則に従っているか
- [ ] マジックナンバーが定数化されているか
- [ ] コードの重複がないか（DRY原則）
- [ ] 適切なコメントが記載されているか（公開API、複雑なロジック）
- [ ] エラーハンドリングが適切に実装されているか

### 2. セキュリティ

- [ ] SQLインジェクション対策が実装されているか
- [ ] XSS対策が実装されているか（フロントエンド）
- [ ] CSRF対策が実装されているか
- [ ] 認証・認可が適切に実装されているか
- [ ] 機密情報（APIキー、パスワード等）がハードコードされていないか
- [ ] 入力値のバリデーションが実装されているか
- [ ] ログに機密情報が出力されていないか

### 3. パフォーマンス

- [ ] N+1クエリが発生していないか
- [ ] 不要なループや再帰がないか
- [ ] 適切なインデックスが設定されているか（DB）
- [ ] キャッシュが適切に使用されているか
- [ ] 非同期処理が適切に実装されているか

### 4. テスト

- [ ] 単体テストが実装されているか
- [ ] テストカバレッジが基準を満たしているか（プロジェクト基準に従う）
- [ ] エッジケースがテストされているか
- [ ] モック/スタブが適切に使用されているか

### 5. ドキュメント

- [ ] READMEが更新されているか（新機能追加時）
- [ ] APIドキュメントが更新されているか（API変更時）
- [ ] 変更履歴（CHANGELOG）が更新されているか
- [ ] コメントが適切に記載されているか

### 6. 依存関係

- [ ] 新規依存パッケージのライセンスが確認されているか
- [ ] 依存パッケージのバージョンが固定されているか
- [ ] セキュリティ脆弱性がないか（npm audit, go mod, 等）

## レビュー出力形式

レビュー結果は以下の形式で出力します：

```
## コードレビュー結果

### ✅ 合格項目
- [項目名]: [詳細]

### ⚠️ 警告項目
- [項目名]: [詳細と推奨修正]

### ❌ 不合格項目（必須修正）
- [項目名]: [詳細と修正方法]

### 📝 改善提案
- [項目名]: [任意の改善提案]
```

## 注意事項

- 不合格項目がある場合は、修正を完了するまでマージを承認しない
- 警告項目は可能な限り修正を推奨するが、緊急時は例外を許可
- 改善提案は任意だが、可能であれば対応を推奨

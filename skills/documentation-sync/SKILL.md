# Documentation Sync Skill

コード変更に伴うドキュメントの同期を強制するスキルです。
API変更、新機能追加、設定変更時にドキュメントが適切に更新されているかを確認します。

## 使用タイミング

- コード実装時（自動チェック）
- コードレビュー時（必須チェック）
- API変更時
- 新機能追加時
- 設定変更時

## チェック項目

### 1. README.md

- [ ] 新機能の説明が追加されているか
- [ ] セットアップ手順が更新されているか（依存関係追加時）
- [ ] 使用方法の例が更新されているか（API変更時）
- [ ] 環境変数の説明が更新されているか（新規追加時）
- [ ] トラブルシューティング情報が更新されているか（必要に応じて）

### 2. APIドキュメント

- [ ] 新規エンドポイントのドキュメントが追加されているか
- [ ] 既存エンドポイントの変更が反映されているか（リクエスト/レスポンス形式）
- [ ] 認証・認可要件が記載されているか
- [ ] エラーレスポンスの説明が記載されているか
- [ ] 使用例が記載されているか

### 3. CHANGELOG.md

- [ ] 変更内容が適切なセクションに追加されているか（Added/Changed/Fixed/Removed）
- [ ] 変更内容が明確に記述されているか
- [ ] 破壊的変更（Breaking Changes）が明記されているか
- [ ] バージョン番号が適切に更新されているか（SemVer）

### 4. コードコメント

- [ ] 公開API（exported functions/methods）にdoc commentが記載されているか
- [ ] 複雑なロジックに説明コメントが記載されているか
- [ ] パラメータと戻り値の説明が記載されているか
- [ ] 使用例が記載されているか（必要に応じて）

### 5. 設定ファイル

- [ ] 設定ファイルの変更がドキュメント化されているか
- [ ] デフォルト値の変更が明記されているか
- [ ] 非推奨設定の移行手順が記載されているか

### 6. アーキテクチャドキュメント

- [ ] アーキテクチャの変更が反映されているか（大幅な変更時）
- [ ] データベーススキーマの変更が記載されているか（マイグレーション時）
- [ ] 依存関係の変更が記載されているか

## ドキュメント更新の優先度

### 必須（マージ前に更新）

- APIドキュメント（API変更時）
- CHANGELOG.md（すべての変更）
- README.md（セットアップ手順の変更時）

### 推奨（可能な限り更新）

- コードコメント（公開API）
- アーキテクチャドキュメント（大幅な変更時）

## 出力形式

```
## ドキュメント同期チェック結果

### ✅ 更新済み
- [ドキュメント名]: [更新内容]

### ⚠️ 更新が必要
- [ドキュメント名]: [更新すべき内容と理由]

### 📝 推奨更新
- [ドキュメント名]: [推奨される更新内容]
```

## ドキュメントテンプレート

### APIエンドポイント

```markdown
### POST /api/v1/users

**説明**: 新規ユーザーを作成します。

**認証**: 必須（Bearer Token）

**リクエスト**:
```json
{
  "name": "string",
  "email": "string"
}
```

**レスポンス**:
- 201 Created: ユーザー作成成功
- 400 Bad Request: バリデーションエラー
- 401 Unauthorized: 認証エラー
```

### CHANGELOGエントリ

```markdown
## [1.2.0] - 2024-01-15

### Added
- 新機能の説明

### Changed
- 変更内容の説明

### Fixed
- 修正内容の説明

### Removed
- 削除内容の説明（破壊的変更の場合は Breaking Changes セクションに記載）
```

## 注意事項

- ドキュメントはコードと同期させることを最優先とする
- ドキュメントの更新漏れは技術的負債となるため、早期に修正する
- 自動生成されるドキュメント（JSDoc, GoDoc等）も定期的に確認する

# Quality Gate Skill

コード品質の最低基準を保証するための品質ゲートスキルです。
マージ前に必須の品質チェックを実行し、基準を満たさないコードのマージを防止します。

## 使用タイミング

- コードレビュー時（必須チェック）
- マージ前（最終チェック）
- CI/CDパイプライン（自動チェック）

## 品質基準

### 1. コード品質メトリクス

- [ ] 循環的複雑度が基準値以下か（プロジェクト基準に従う、推奨: 10以下）
- [ ] コード重複率が基準値以下か（推奨: 5%以下）
- [ ] 関数/メソッドの行数が基準値以下か（推奨: 50行以下）
- [ ] ファイルサイズが基準値以下か（推奨: 500行以下）

### 2. テストカバレッジ

- [ ] 単体テストカバレッジが基準値を満たしているか（プロジェクト基準に従う、推奨: 80%以上）
- [ ] 新規コードのテストカバレッジが基準値を満たしているか
- [ ] クリティカルパスのテストが実装されているか

### 3. リント・フォーマット

- [ ] リントエラーが0件か（警告は許容する場合あり）
- [ ] コードフォーマットが統一されているか（gofmt, prettier, black等）
- [ ] 未使用のインポート/変数が削除されているか

### 4. 型安全性

- [ ] TypeScriptの型エラーが0件か（TypeScript使用時）
- [ ] 型アサーションが適切に使用されているか
- [ ] any型の使用が最小限か（TypeScript使用時）

### 5. パフォーマンス

- [ ] 明らかなパフォーマンス問題がないか（N+1クエリ、不要なループ等）
- [ ] メモリリークの可能性がないか
- [ ] 適切なインデックスが使用されているか（DBクエリ）

### 6. 保守性

- [ ] コードが読みやすいか（命名、構造、コメント）
- [ ] 適切な抽象化が行われているか
- [ ] 依存関係が適切に管理されているか

## 品質ゲートの判定

### 合格条件（マージ可能）

すべての必須項目が基準を満たしていること：
- リントエラー: 0件
- テストカバレッジ: 基準値以上
- 重大なセキュリティ問題: 0件
- 重大なパフォーマンス問題: 0件

### 不合格条件（マージ不可）

以下のいずれかに該当する場合：
- リントエラーが存在する
- テストカバレッジが基準値未満
- Critical/Highレベルのセキュリティ問題が存在する
- 重大なパフォーマンス問題が存在する

### 警告条件（マージ可能だが改善推奨）

以下の項目に該当する場合：
- コード重複率が基準値を超えている
- 循環的複雑度が基準値を超えている
- Medium/Lowレベルのセキュリティ問題が存在する

## 出力形式

```

### ✅ 合格
すべての品質基準を満たしています。マージ可能です。

### ❌ 不合格
以下の項目が基準を満たしていません。修正が必要です。

#### 必須修正項目
- [項目名]: [詳細と修正方法]

#### 警告項目（改善推奨）
- [項目名]: [詳細と改善提案]

### 📊 メトリクス
- テストカバレッジ: [XX]% (基準: [XX]%)
- コード重複率: [XX]% (基準: [XX]%以下)
- 循環的複雑度: [平均値] (基準: [XX]以下)
- リントエラー: [XX]件
```

## プロジェクト固有の設定

品質基準はプロジェクトごとに異なる場合があります。
プロジェクト固有の基準は `.claude/project/quality-gate.md` に定義してください。

### デフォルト基準（参考）

```yaml
code_quality:
  cyclomatic_complexity: 10
  code_duplication: 5%
  max_function_lines: 50
  max_file_lines: 500

test_coverage:
  minimum: 80%
  critical_path: 100%

lint:
  errors: 0
  warnings: allowed
```

## 注意事項

- 品質ゲートは開発速度と品質のバランスを考慮する
- 緊急時は例外を許可するが、後で必ず修正する
- 品質基準は定期的に見直し、チームの成長に合わせて調整する

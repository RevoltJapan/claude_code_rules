# Run Command（実装〜レビュー反復）

対象タスク: $ARGUMENTS

## 実行プロセス

1. **実装フェーズ**（`impl` と同等）
   - タスクを読み、計画を立て、段階的に実装する
2. **検証フェーズ**
   - `quality-check` を適用し、未充足の受入条件を洗い出す
3. **修正フェーズ**
   - 指摘を反映し、再度 `quality-check` で確認する
4. 受入条件のチェックリストがすべて完了するまで 2〜3 を繰り返す

## 注意

- 自動ループは実装時間・トークン消費が増える。並列開発（worktree 等）前提で運用する
- 無限ループを避けるため、同一指摘が繰り返す場合はユーザーに判断を仰ぐ

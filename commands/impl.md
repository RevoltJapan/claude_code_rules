# Implementation Command

実装タスク: $ARGUMENTS

## 実行プロセス

### Phase 1: 理解と計画
1. タスクファイルを完全に読み込む
2. 参照ドキュメント（architecture.md等）を確認
3. 現状のコードベースを把握
4. 実装計画を立て、ユーザーに確認を求める

### Phase 2: 段階的実装
5. チェックリストの各項目を順番に実装
6. 各ステップ完了後、動作確認を実施
7. 問題があれば即座に報告し、次に進む前に修正

### Phase 3: 検証
8. すべての受入条件を満たしているか確認
9. 影響範囲の確認
10. テストの実行

## 実装ルール
- 一度に多くを変更しない（最大5ファイル程度）
- 各ステップでコミット可能な状態を保つ
- architecture.mdの規約を厳守
- 不明点は実装前に必ず質問する

## 品質基準
- TypeScriptの型安全性を保つ
- ESLintエラーがないこと
- ビルドが成功すること
- 既存機能が壊れていないこと
